# AI Product Engineer Roadmap

## 文件用途

把产品、设计和工程能力组织成长期路线。这里也承接原 AI Product 笔记中的判断框架：技术想法最终要落到用户、场景、价值、风险和验证方法。

## 长期能力地图

1. **AI 产品判断**：识别真正需要 AI 的问题，定义质量、风险和 Human in the Loop。
2. **Agent 系统理解**：掌握 Agent Loop、Tool、Skill、MCP、Context、Memory、Plan 与 Multi-Agent。
3. **产品工程实现**：使用 TypeScript、React、Next.js 和 AI SDK 完成可运行原型。
4. **源码与系统分析**：能阅读优秀开源项目，理解生产架构的取舍。
5. **实验与评测**：用小实验、Evals 和用户反馈验证产品假设。

## 核心问题

- 什么问题真的需要 AI，而不是普通自动化？
- 如何定义“足够好”的输出质量和失败边界？
- 用户应在哪些节点确认、接管或撤销？
- 怎样把学习转化为真实 Demo 与工作改进？

## 阶段路线

### 阶段一：建立共同语言

补齐术语、Agent 主循环、Tool / Skill / MCP、Context / Memory 和 TypeScript 基础。

### 阶段二：做出完整小产品

用 Next.js 和 AI SDK 完成一个包含流式输出、工具调用、错误恢复和基础评测的 Demo。

### 阶段三：进入生产级问题

研究权限、安全、压缩、记忆、多 Agent、可观测性和长期任务，并在工作产品中验证。
