# Multi-Agent

## 主题说明
理解多个 Agent 如何分工、并行、共享信息和汇总结果，以及何时它优于单 Agent。

## 核心概念
主 Agent、子 Agent、任务委派、上下文隔离、结果汇总、冲突与成本。

## 我需要理解的问题
- [ ] 哪类任务适合拆给多个 Agent？
- [ ] 子 Agent 应继承哪些上下文？
- [ ] 多个结果冲突时由谁判断？

## 和产品工作的关系
用户关心的是结果是否更快、更可靠，而不是背后有多少角色；界面仍应保留清晰主线。
