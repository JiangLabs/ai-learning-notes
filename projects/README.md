# Projects

## 文件用途

管理个人 Demo、产品想法和实验，让知识学习最终落到可以运行、展示或验证的东西上。

## 核心问题

- 这个项目要验证哪一个明确假设？
- 最小版本需要做到什么，不需要做到什么？
- 结果怎样反哺 knowledge、work 或 roadmap？

## 当前文件索引

- [demos.md](demos.md)：可展示、可运行的 Demo 清单
- [ideas.md](ideas.md)：尚未进入实现的产品想法
- [experiments.md](experiments.md)：小范围技术与产品实验

## 学习 / 记录模板

问题 → 假设 → 最小范围 → 实现 → 结果 → 结论 → 下一步。

## 后续更新方向

- 每月至少选择一个小想法进入实验。
- 为完成的 Demo 补充截图、代码仓库、复盘和可复用组件。
